Some times we may choose not to undergo all the various steps and tedious processes of storing our form data to a database. To escape from that and still have our form data securly stored, we can use google sheet to do that.In this blog, we will walk through the steps from creating an html form to storing the the form input data to a google sheet.

Creating an html form

Firstly, we need to create a project folder in file explorer then use any text editor of our choice and open the folder.While at the open folder in text editor, create a file called index.html or name it any name you want but with the html ending.Below is a simple form with two input fields and a submit button.

Setting up a google sheet

  1. lets us go to google sheet and create a new sheet
  2. Make sure the header of the sheet correspond to the number of input fields in the form with exact name and spellings

Creating a google app script

On your own screen on the above video, click on extensions → App scripts.A new google scripts will be opened.Give it a suitable name.

After giving the script a name, replace the default function on the scripts with the code on the video below

Make sure to save the code before proceeding to Running the initial setup

Running the initial Setup

Ones you press the run button in the App script interface, a pop-up box will show up for you to review permission.Click the review permission button to get to the next screen.Since the scripts has not been reviewed by google, it will generate a warning message.It is required to click "Go to form12(unsafe)"(form12 will be change to your own script name) for the script to have the right permission to update your form

Ones you have given your form the right permission,the following output will be seen on the screen

At this point your script is having the right permission to continue to the next step

Adding a trigger for the scripts

Ones trigger is click from the window above, select Add triggers on the sidebars.
Then you select the following from the window that appear.

  • Choose which to run

    • doPost

  • Choose which deployment should run

    • Head

  • Select event source

    • From spreadsheet

  • Select event type

    • On from Submit

When you are done with that, select save

Deploy the project

At this point your project is ready to be published.

As it is in the screen above, on your own screen, open the deploy button and select new deployment from the the dropdown list.Click the select type button , then select web app

On the next window, fill in the fields as shown on the screen below

You can write anything in the "new description" section.Click the deploy button.

Note: Before moving to the next step , copy and save the wep app URL
